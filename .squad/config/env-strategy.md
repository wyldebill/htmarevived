# Environment Variable Strategy — Google Maps & App Config

**Last Updated:** 2026-04-18  
**Scope:** MVP — Build-time and runtime configuration  
**Target Platforms:** Android, iOS (Flutter)

---

## Overview

This document defines how environment variables are consumed at build-time and runtime for Google Maps API key configuration and app-level settings. The strategy supports local development, CI/CD, and production deployment.

---

## Environment Variables

### Primary Configuration

| Variable | Purpose | Scope | Required | Example |
|---|---|---|---|---|
| `GOOGLE_MAPS_API_KEY` | Google Maps Platform API key (mobile) | Build-time + Runtime | Yes | `AIzaSyD...` |
| `FLUTTER_ENV` | Environment indicator (dev/staging/prod) | Runtime | No | `dev` |
| `FIREBASE_CONFIG` | Firebase config JSON (passed at build time) | Build-time | Yes | Provided by user |

---

## Build-Time Configuration

### Android

1. **Method:** Store API key in `local.properties` and reference via `build.gradle`
2. **File Path:** `android/local.properties`
3. **Configuration:**
   ```properties
   google.maps.api.key=YOUR_API_KEY_HERE
   ```

4. **gradle Integration:** Reference in `android/app/build.gradle`:
   ```gradle
   android {
       ...
       buildTypes {
           release {
               ...
               manifestPlaceholders = [
                   googleMapsApiKey: google.maps.api.key
               ]
           }
       }
   }
   ```

5. **AndroidManifest.xml:** Already includes placeholder for API key in meta-data tag
   ```xml
   <meta-data
       android:name="com.google.android.geo.API_KEY"
       android:value="${googleMapsApiKey}" />
   ```

### iOS

1. **Method:** Add API key via CocoaPods and build phase
2. **File Path:** Environment variable passed to build system
3. **Configuration:**
   ```bash
   export GOOGLE_MAPS_API_KEY="YOUR_API_KEY_HERE"
   flutter build ios --build-name=1.0.0
   ```

4. **Runner.xcconfig Integration:** Build settings will automatically use the env var
5. **Info.plist:** Contains placeholder for runtime injection

## Runtime Configuration

### Flutter App Initialization

1. **Load Point:** `main.dart` — called before Firebase initialization
2. **Method:** Read environment variable and pass to GoogleMapsWidget
3. **Implementation Pattern:**
   ```dart
   const String googleMapsApiKey = String.fromEnvironment('GOOGLE_MAPS_API_KEY');
   
   void main() {
     if (googleMapsApiKey.isEmpty) {
       // Log warning and show user-friendly error
       print('WARNING: GOOGLE_MAPS_API_KEY not set');
     }
     runApp(MyApp());
   }
   ```

4. **Fallback Behavior:** If API key is missing, the app will start but map screens will display a clear error message instead of crashing.

## Local Development Setup

### Prerequisites

1. Create a `.env` file in the repo root (gitignored)
2. Add your Google Maps API key:
   ```
   GOOGLE_MAPS_API_KEY=YOUR_API_KEY_HERE
   FLUTTER_ENV=dev
   ```

### Running Locally

**Android (Emulator):**
```bash
export GOOGLE_MAPS_API_KEY=$(grep GOOGLE_MAPS_API_KEY .env | cut -d '=' -f2)
flutter run --dart-define=GOOGLE_MAPS_API_KEY=$GOOGLE_MAPS_API_KEY
```

**iOS (Simulator):**
```bash
export GOOGLE_MAPS_API_KEY=$(grep GOOGLE_MAPS_API_KEY .env | cut -d '=' -f2)
flutter run --dart-define=GOOGLE_MAPS_API_KEY=$GOOGLE_MAPS_API_KEY
```

**Or use Flutter's built-in `--dart-define-from-file`:**
```bash
flutter run --dart-define-from-file=.env
```

## CI/CD Configuration (GitHub Actions)

### Storing Secrets

1. Navigate to **Settings > Secrets and variables > Actions**
2. Add these repository secrets:
   - `GOOGLE_MAPS_API_KEY` – your actual Google Maps API key
   - `FIREBASE_CONFIG` – Firebase configuration JSON (if needed)

### Workflow Usage

In `.github/workflows/build.yml`:
```yaml
- name: Build Flutter App
  env:
    GOOGLE_MAPS_API_KEY: ${{ secrets.GOOGLE_MAPS_API_KEY }}
  run: |
    flutter build apk --dart-define=GOOGLE_MAPS_API_KEY=$GOOGLE_MAPS_API_KEY
    flutter build ios --dart-define=GOOGLE_MAPS_API_KEY=$GOOGLE_MAPS_API_KEY
```

## Firebase Configuration

### User-Provided Configuration

1. User will provide Firebase config files (to be determined)
2. These will be placed in project-specific locations:
   - **Android:** `android/app/google-services.json` (generated by Firebase console)
   - **iOS:** `ios/Runner/GoogleService-Info.plist` (generated by Firebase console)

3. At runtime, these are loaded by FlutterFire:
   ```dart
   void main() async {
     WidgetsFlutterBinding.ensureInitialized();
     await Firebase.initializeApp();
     runApp(MyApp());
   }
   ```

### Firebase Realtime Database URL

- Will be provided by the user in Firebase config
- Accessed through `FirebaseDatabase.instance` (automatically uses the configured URL)
- No additional environment variable needed once Firebase is initialized

## Error Handling & Fallbacks

### Missing API Key

**Scenario:** `GOOGLE_MAPS_API_KEY` env var not set

**Behavior:**
1. App starts normally
2. Map screens show error banner: "Maps are temporarily unavailable. Check your configuration."
3. List and detail screens work normally
4. No crash or broken UI

### Missing Firebase Config

**Scenario:** Firebase config files not found

**Behavior:**
1. App startup halts with clear error dialog
2. Message: "Firebase configuration missing. Please provide firebase config files."
3. Developer instructed to check setup documentation

### Network/API Errors

**Scenario:** API key is invalid or quota exceeded

**Behavior:**
1. Map displays error overlay
2. User sees: "Unable to load map. Please try again."
3. Retry button provided
